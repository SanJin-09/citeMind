# citeMind 前端交互与使用体验修复计划

> 审查日期：2026-06-21  
> 审查范围：`apps/desktop/src/renderer`、桌面窗口约束、现有主界面截图与开发环境启动结果  
> 目标：不改变 citeMind 的核心产品方向，优先修复影响任务完成率的问题，再补齐反馈、引导、动画、可访问性和视觉一致性。

## 1. 结论摘要

citeMind 已具备完整的业务能力，但当前前端更接近“功能控制台”，还没有形成稳定、可预期的桌面产品体验。主要问题不是单纯缺少动画，而是以下五类体验债务叠加：

1. **首启缺少引导**：用户必须先配置 Ark API Key，但界面不会主动展示完成路径，通常要在首次提问失败后才知道前置条件。
2. **长任务缺少过程反馈**：导入、解析、OCR、Embedding、索引等后台任务已有事件能力，主界面却没有持续进度、完成通知和恢复入口。
3. **存在无效或不完整交互**：来源筛选框无筛选逻辑，来源菜单与头像按钮无动作；在已有对话时执行全局搜索，结果被状态更新但不会显示。
4. **复杂功能直接暴露技术实现**：MCP、Tool、LangGraph、FTS5、LanceDB、Schema 等术语占据主要界面，增加普通用户理解成本。
5. **交互基础设施不足**：七类弹窗缺少焦点锁定、Escape 关闭、初始焦点和关闭后焦点恢复；全局缺少统一通知、加载骨架、空状态和动画规范。

建议按以下顺序修复：

```text
任务可完成性
    ↓
状态与错误可见
    ↓
首启引导与空状态
    ↓
布局、键盘与可访问性
    ↓
动效与视觉打磨
```

不建议先做大规模视觉换肤。若不先修复状态模型和任务路径，增加动画只会让现有问题显得更精致，并不会提升使用成功率。

## 2. 审查依据与现状

### 2.1 前端结构

- `App.tsx`：约 6315 行，集中管理页面布局、数据加载、全部业务状态、弹窗和辅助函数。
- `styles.css`：约 3299 行，采用单文件全局样式。
- React 组件已覆盖主工作区、对话、引用、Agent Trace、写作、来源维护、外部资料、设置等功能。
- 当前未引入路由、状态管理、组件库或动画库。
- Electron 窗口最小尺寸为 `960 × 640`。

### 2.2 运行检查

开发环境可以成功完成 Electron Main、Preload 和 Renderer 构建，并启动应用。自动化浏览器对本地 URL 的访问被运行环境安全策略拦截，因此动态检查未覆盖真实点击链路；本报告中的交互判断来自：

- 当前 React 事件与状态代码；
- CSS 布局和响应式规则；
- `images/main_screen.png` 现有主界面截图；
- README 中描述的核心任务流；
- Electron 窗口配置和开发启动日志。

### 2.3 当前主要用户路径

```text
首次启动
  → 配置 Ark API
  → 创建/选择知识库
  → 导入文件、网页或外部资料
  → 等待解析与索引
  → 选择参与检索的来源
  → 提问
  → 查看 Agent 执行过程
  → 阅读带引用回答
  → 查看证据或导出
  → 进入复习/写作工作流
```

当前路径中，最薄弱的环节是“首次启动”“等待处理”“错误恢复”和“引用跳转”。

## 4. 目标交互设计

### 4.1 首次启动与新手引导

首次启动不应直接展示一个看似可用、实际会在提交时失败的输入框。建议使用“渐进式启用”：

```text
┌──────────────────────────────────────────────────────────────┐
│ citeMind                                      [系统诊断]      │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  开始建立你的可验证知识库                                    │
│                                                              │
│  ① 配置模型服务       ○ 未完成        [配置 Ark API →]       │
│  ② 添加第一份资料     ○ 等待上一步                           │
│  ③ 等待资料可检索     ○ 尚未开始                             │
│  ④ 提出第一个问题     ○ 尚未开始                             │
│                                                              │
│  你的资料保存在本机；回答会附带可追踪引用。                  │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

规则：

- 仅首次启动或当前知识库为空时展示。
- 每一步完成后使用 180–240ms 状态过渡，不弹出打断式庆祝动画。
- 用户可以跳过解释，但不能跳过真实前置条件。
- 配置失败时在当前步骤内展示原因和重试，不把用户丢回主界面。

### 4.2 主工作区信息架构

保留三栏结构，但明确每栏职责，降低顶部按钮密度：

```text
┌──────────────┬────────────────────────────────┬───────────────┐
│ 来源         │ 对话 / 搜索                    │ 证据          │
│              │                                │               │
│ [＋添加资料] │ [搜索当前知识库………………]       │ 引用 [2]      │
│ [筛选……  ×] │                                │ 来源名称      │
│              │ 用户问题                       │ 页码/段落     │
│ ☑ 文档 A     │                                │               │
│ ◔ 文档 B 63% │ citeMind 正在检索 8 个候选…    │ 原文片段      │
│ ! 文档 C     │ ├ 已读取 3 个来源              │               │
│              │ └ 正在校验引用                 │ [打开原文]    │
│ [任务 2]     │                                │ [来源列表定位]│
│              │ [输入问题…………………] [发送]    │               │
└──────────────┴────────────────────────────────┴───────────────┘
```

关键变化：

- 全局搜索与对话并存，搜索结果使用对话区顶部可关闭面板，不依赖“是否已有消息”。
- 来源卡片直接显示处理状态和进度，失败状态提供重试。
- 证据区将“打开原文”和“定位来源”拆为两个动作。
- 后台任务入口固定在来源栏底部或底部状态栏，不再隐藏在状态文本中。

### 4.3 导入与后台任务

文件选择后立即创建可见任务，不等待整个导入调用结束：

```text
┌─ 后台任务 ────────────────────────────────────────┐
│ 正在处理 2 项                              [收起] │
│                                                   │
│ 产品说明.pdf                                      │
│ 解析完成  →  分块完成  →  Embedding 63%          │
│ ███████████████░░░░░░  00:41          [暂停][取消]│
│                                                   │
│ 截图.png                                          │
│ OCR 失败：未识别到有效文字             [重试][详情]│
└───────────────────────────────────────────────────┘
```

状态要求：

- `pending`：排队中；
- `running`：显示当前阶段、进度、耗时；
- `paused`：显示继续；
- `retrying`：显示重试次数和下一步；
- `failed`：显示可操作错误；
- `completed`：保留短时成功状态，然后进入历史。

任务中心应直接复用已有 `BackgroundJobRecord`，不需要创建第二套前端任务模型。

### 4.4 对话生成状态

Agent Trace 是产品差异点，但当前信息密度较高。默认层级应为：

```text
运行中：正在校验引用 · 00:18                         [展开]
```

展开后再显示：

```text
✓ 检索证据       找到 8 个候选片段
✓ 读取来源       使用 3 个来源
● 校验引用       正在检查第 2/4 段
○ 整理回答

[高级执行记录 ▸]   ← Tool、Skill、参数和 stdout 放在这里
```

这样既保留可审计性，也避免普通用户先看到 Tool 和内部阶段名称。

### 4.5 设置页

```text
┌─ 设置 ────────────────────────────────────────────┐
│ [API 与模型] [用量] [存储与维护]                  │
├───────────────────────────────────────────────────┤
│ Ark API Key                         已配置 ✓       │
│ ••••••••••••••••                  [替换密钥]      │
│                                                   │
│ 默认回答模型     Seed Lite          [选择 ▾]       │
│ 默认向量模型     Seed Embedding     [选择 ▾]       │
│                                                   │
│ 模型状态：3 项可用                                │
│                                      [重新验证]   │
└───────────────────────────────────────────────────┘
```

默认不展示 Base URL、safeStorage、模型 ID 和质量指标；这些信息移动到“高级诊断”折叠区。

## 5. 统一交互规范

### 5.1 反馈层级

| 层级     | 使用场景                   | 展示方式                     |
| -------- | -------------------------- | ---------------------------- |
| 字段反馈 | URL 无效、名称为空         | 字段下方即时提示             |
| 区域反馈 | 搜索失败、章节保存失败     | 当前区域内提示，附重试       |
| 全局通知 | 导出成功、索引完成         | 右上角 Toast，可进入通知历史 |
| 持续任务 | 导入、OCR、索引、写作生成  | 任务中心                     |
| 阻断确认 | 删除知识库、密钥、清理数据 | Modal，明确影响范围          |
| 系统故障 | Worker 离线、存储不可用    | 顶部常驻 Banner，不自动消失  |

禁止同一个错误同时出现在两个层级。

### 5.2 加载规范

- 小于 400ms：不展示加载态，避免闪烁。
- 400ms–2s：按钮 Spinner 或局部骨架。
- 超过 2s：展示阶段说明和可取消能力。
- 无法估算进度：使用循环进度条，但必须显示当前阶段。
- 已有内容刷新：保留旧内容并显示“正在刷新”，不要清空后重绘。

### 5.3 动效规范

当前 CSS 只有少量 `transition`，没有 `@keyframes` 和 `prefers-reduced-motion`。建议建立轻量规范，不引入动画库：

| 场景              |        时长 | 效果                              |
| ----------------- | ----------: | --------------------------------- |
| Hover/按压        |   100–140ms | 颜色、边框、1px 位移              |
| 菜单/Popover      |   140–180ms | 透明度 + 4px 位移                 |
| 抽屉/证据面板     |   180–240ms | 位移 + 透明度                     |
| Modal             |   180–220ms | 背景渐入，内容缩放 `0.98 → 1`     |
| 列表新增/状态更新 |   180–240ms | 淡入，不改变阅读位置              |
| 成功状态          |       200ms | 图标描边/颜色变化，不使用大幅弹跳 |
| Skeleton          | 1200–1600ms | 低对比度 shimmer                  |

统一缓动：

```css
--ease-standard: cubic-bezier(0.2, 0, 0, 1);
--ease-emphasized: cubic-bezier(0.2, 0.8, 0.2, 1);
```

必须增加：

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    scroll-behavior: auto !important;
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```
