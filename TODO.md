# citeMind 开发任务清单

> 依据：`rag-personal-knowledge-base-product-and-architecture.md`
> 目标：先完成可验证、可追溯的个人知识库 MVP，再扩展可靠性与写作能力。
> 标记：`P0` 为 MVP 阻塞项，`P1` 为 MVP 完整项，`P2` 为第二阶段。

## 0. 开工前确认

- [x] `P0` 首批仅支持 macOS Apple Silicon，使用 electron-builder 生成 arm64 安装包。
- [x] `P0` 采用可版本化的精简 Seed 模型目录，并通过实际轻量调用探测当前 Ark API Key 的可用能力：
  - 默认对话：`doubao-seed-2-0-lite-260428`。
  - 高质量对话：`doubao-seed-2-0-pro-260215`。
  - Embedding：`doubao-embedding-vision-251215`，首版固定使用 1024 维 Dense Vector。
- [x] `P0` 确认解析与 OCR 方案，解析失败必须可见，不允许静默跳过：
  - PDF、DOCX、图片以 Docling 为主。
  - 网页优先使用正文提取器，Playwright 仅作为动态网页兜底。
  - PDF 样例覆盖数字版、扫描版、多栏和含表格文档。
  - DOCX 样例覆盖标题层级、段落和表格。
  - 图片样例覆盖截图和拍照文本。
  - 网页样例覆盖静态网页和需要 Playwright 的动态网页。
- [x] `P0` 确认“点击引用跳转原文”的首版定位方式：
  - PDF：页码 + bounding box 高亮。
  - DOCX：标题路径 + 段落锚点。
  - 网页：标题路径 + 快照文本块。
  - 图片：图片预览 + OCR 区域高亮。
- [x] `P0` 本地数据与备份使用系统应用数据目录：
  - 通过 Electron `app.getPath("userData")` 获取根目录，不写死绝对路径。
  - 原始文件、网页快照、解析结果、SQLite 和 LanceDB 默认保存于该目录。
  - 数据库结构升级前自动创建 SQLite 快照；LanceDB 索引视为可重建数据。
  - 使用数据库迁移管理版本升级，迁移失败时保留升级前快照并停止写入。

## 1. 工程底座

### 1.1 桌面端工程

- [x] `P0` 初始化 Electron + React + TypeScript + Vite 工程。
- [x] `P0` 配置代码检查、格式化、单元测试和构建脚本。
- [x] `P0` 开启 `contextIsolation`，关闭 `nodeIntegration`。
- [x] `P0` 建立受限 Preload IPC 接口，禁止 Renderer 直接访问密钥和文件系统。
- [x] `P0` 实现 Electron Main 对 Python Worker 的启动、健康检查、退出与异常重启。
- [x] `P0` 实现 JSON-RPC over stdio 通信，支持请求、响应、通知、超时和错误码。

### 1.2 Python Worker 工程

- [x] `P0` 初始化 Python 3.12 工程、依赖管理、测试和日志配置。
- [x] `P0` 定义 JSON-RPC 方法注册、参数校验和统一错误结构。
- [x] `P0` 定义核心接口：`DocumentRepository`、`IngestionPipeline`、`IndexVersionManager`、`HybridRetriever`、`CitationValidator`、`BackgroundJobManager`。
- [x] `P0` 定义 `ModelGateway`，隔离 Ark SDK、LangChain 与业务层。
- [x] `P0` 确保日志不记录 API Key 和完整敏感文档正文。

### 1.3 本地存储

- [ ] `P0` 建立 SQLite 数据库迁移机制。
- [ ] `P0` 创建首版必要表：KnowledgeBase、Source、SourceVersion、Chunk、IndexVersion、SeedApiCredential、ModelCapability、Conversation、Message、AnswerCitation、BackgroundJob；允许非 PDF 来源的 `page_number` 为空。
- [ ] `P0` 配置 SQLite FTS5，并验证中文关键词检索效果。
- [ ] `P0` 初始化 LanceDB，定义向量表与 `index_version` 隔离规则。
- [ ] `P0` 基于 Electron `app.getPath("userData")` 实现原始文件、网页快照、解析产物、SQLite 和 LanceDB 的本地目录管理。

**里程碑验收**

- [ ] Electron 可稳定启动并调用 Python Worker 的健康检查接口。
- [ ] 数据库可迁移、可重建，FTS5 和 LanceDB 均可完成最小读写测试。
- [ ] Renderer 无法直接读取 Ark API Key。

## 2. Seed API 配置与模型网关

- [ ] `P0` 使用 Electron `safeStorage` 加密保存和读取 Ark API Key。
- [ ] `P0` 内置已确认的 3 项 Seed 模型目录，目录配置可版本化，不依赖 Ark API Key 自动枚举全部模型。
- [ ] `P0` 实现对话、Embedding、视觉和结构化输出的轻量能力探测。
- [ ] `P0` 展示可调用、未开通、无权限、限流、调用失败等验证状态。
- [ ] `P0` 实现 Ark SDK 模型网关：流式回答、结构化生成、Embedding、超时、重试和用量统计。
- [ ] `P0` 结构化输出探测失败时，降级为 JSON Prompt，并执行解析校验。
- [ ] `P0` 实现 Seed API 设置页和模型选择页。
- [ ] `P0` 为模型验证、密钥隔离和错误映射补充自动化测试。

**里程碑验收**

- [ ] 用户可保存 Ark API Key、验证指定模型并看到明确状态。
- [ ] React Renderer 全程接触不到明文 API Key。
- [ ] 对话与 Embedding 模型可通过统一 `ModelGateway` 调用。

## 3. 知识库与导入闭环

### 3.1 知识库管理

- [ ] `P0` 实现知识库的新建、重命名、切换和删除确认。
- [ ] `P0` 实现知识库来源列表及处理状态汇总。
- [ ] `P0` 确保文档、索引、对话均按知识库隔离。

### 3.2 持久化任务基础

- [ ] `P0` 实现 BackgroundJob 状态机：pending、running、completed、paused、cancelled、failed、retrying。
- [ ] `P0` 保存任务进度、检查点、重试次数和错误信息。
- [ ] `P0` 实现任务进度通知，使前端可实时展示解析、OCR、Embedding 和索引状态。
- [ ] `P1` 应用重启后扫描未完成任务，并支持继续或重新执行。
- [ ] `P1` 实现暂停、取消和失败重试入口。

### 3.3 文档解析与检查

- [ ] `P0` 实现 PDF 导入与 Docling 解析，保留页码、标题路径和可用的 bounding box。
- [ ] `P0` 实现 DOCX 导入与 Docling 结构解析，生成标题路径和段落锚点。
- [ ] `P0` 实现图片导入与 Docling OCR，保留可用于高亮的 OCR 区域。
- [ ] `P0` 实现网页链接抓取、正文提取与快照保存，动态网页失败时使用 Playwright 兜底。
- [ ] `P0` 标准化解析结果，保留原文与标准化正文。
- [ ] `P0` 实现解析检查页：成功、需要 OCR、失败、重复、错误原因和解析预览。
- [ ] `P0` 文档完整索引前保持“处理中”，不得进入正式检索结果。

### 3.4 分块与索引

- [ ] `P0` 定义带版本号的结构化分块策略。
- [ ] `P0` 为每个 Chunk 保存来源、来源版本、可选页码、可选 bounding box、标题路径、段落或快照锚点、原文、标准化正文和内容 Hash。
- [ ] `P0` 将文本块写入 SQLite 元数据与 FTS5。
- [ ] `P0` 批量调用 Embedding，并将向量写入 LanceDB。
- [ ] `P0` 索引完成后原子更新文档版本为“可检索”。
- [ ] `P0` 为 PDF、DOCX、图片、网页各准备至少一个端到端导入测试。

**里程碑验收**

- [ ] 四类来源均可导入，用户能看到处理进度、解析结果和失败原因。
- [ ] 已完成索引的文本块可同时被 FTS5 和 LanceDB 检索。
- [ ] 应用重启不会损坏已完成的文档与索引。

## 4. 可信检索与问答闭环

### 4.1 混合检索

- [ ] `P0` 实现限定当前知识库和当前有效索引版本的 FTS5 关键词检索。
- [ ] `P0` 实现限定当前知识库和当前有效索引版本的 LanceDB 向量检索。
- [ ] `P0` 实现两路结果合并、去重、排序和上下文组装。
- [ ] `P0` 返回关键词命中、语义相关度和检索排名等可解释信息。
- [ ] `P1` 预留可选重排序接口，并记录重排序模型版本。

### 4.2 对话与引用

- [ ] `P0` 实现 Conversation、Message 持久化及流式回答。
- [ ] `P0` Prompt 明确要求证据不足时拒绝自由补全。
- [ ] `P0` 要求模型按“回答段落 + evidence_chunk_ids”输出结构化结果。
- [ ] `P0` 实现 `CitationValidator`：
  - 校验 Chunk 真实存在。
  - 校验 Chunk 来自本次检索候选。
  - 校验来源版本仍有效。
  - 按来源类型校验页码、bounding box、标题路径、段落锚点、快照文本块或 OCR 区域可定位。
  - 校验每个回答段落至少有一条有效证据。
- [ ] `P0` 引用校验失败时删除无效引用、标记证据不足，必要时仅重试一次检索与生成。
- [ ] `P0` 保存回答实际使用的模型、参数、索引版本和生成时间。

### 4.3 问答界面

- [ ] `P0` 实现知识库、对话工作区、来源与证据三栏主界面。
- [ ] `P0` 每段回答展示来源、可用的页码或结构定位、引用原文、证据强度和检索信息，不为非 PDF 来源虚构页码。
- [ ] `P0` 实现点击引用跳转到原文对应位置。
- [ ] `P0` 实现纯关键词/自然语言资料搜索入口。
- [ ] `P0` 覆盖“有充分证据”“证据不足”“模型返回虚假引用”三类端到端测试。

**里程碑验收：可信问答闭环**

- [ ] 用户可导入资料、提问、获得带真实引用的回答并跳转原文。
- [ ] 系统不会向用户展示不属于本次检索结果的引用。
- [ ] 证据不足时明确提示，不生成无依据事实。

## 5. 完成 MVP 必须能力

### 5.1 重复检测

- [ ] `P1` 导入前计算原始文件 SHA-256，识别完全重复文件。
- [ ] `P1` 解析后计算标准化正文 Hash，识别内容重复文件。
- [ ] `P1` 检测到重复时提供：跳过导入、保留独立副本、关联已有文档。
- [ ] `P1` 重复资料不得被自动删除。

### 5.2 模型切换与索引版本

- [ ] `P1` 支持当前对话从下一条消息开始切换对话模型。
- [ ] `P1` 支持设置新对话的默认模型，且不改写已有历史消息。
- [ ] `P1` 上下文窗口不足时使用近期消息与历史摘要，不删除原始对话。
- [ ] `P1` 实现 IndexVersion 生命周期与当前版本指针。
- [ ] `P1` 切换 Embedding 模型前展示文档数、文本块数、调用量和费用提示。
- [ ] `P1` 后台构建新索引版本，完整性验证通过后原子切换。
- [ ] `P1` 短期保留旧索引，并支持回滚、暂停、取消和失败重试。

### 5.3 导出与交付

- [ ] `P1` 支持将回答或对话导出为 Markdown，保留引用信息。
- [ ] `P1` 展示 API 调用量和基础费用提示。
- [ ] `P1` 完成应用数据清理、知识库删除和索引回收流程。
- [ ] `P1` 使用 electron-builder 生成 macOS Apple Silicon arm64 安装包。
- [ ] `P1` 完成安装、升级、卸载、异常退出恢复和数据保留验证。

**MVP 发布验收**

- [ ] 覆盖产品文档中 14 项 MVP 必须功能。
- [ ] 核心流程具备自动化测试，关键失败路径可恢复或给出明确错误。
- [ ] 安全检查通过：密钥隔离、IPC 白名单、日志脱敏、Renderer 安全配置。
- [ ] 使用真实样例完成一次从安装到 Markdown 导出的验收测试。

## 6. 第二阶段：资料维护与写作

### 6.1 网页与文档版本维护

- [ ] `P2` 保存网页 ETag、Last-Modified、正文 Hash、快照和检查时间。
- [ ] `P2` 应用运行期间执行网页更新检查，并生成新版本而非覆盖旧版本。
- [ ] `P2` 展示版本差异，由用户决定是否更新当前索引。
- [ ] `P2` 仅重新处理发生变化的文本块。
- [ ] `P2` 支持用户指定替代文档、复查时间和过期状态。
- [ ] `P2` 模型推测过期或冲突时仅生成待确认建议。

### 6.2 自动分类与关联

- [ ] `P2` 基于文件夹、文件名、标题、作者和时间执行规则分类。
- [ ] `P2` 使用模型生成主题标签建议，并允许用户确认、修改或忽略。
- [ ] `P2` 保存用户修正结果供后续复用。
- [ ] `P2` 实现 Relation 数据：关联类型、依据、置信程度和确认状态。
- [ ] `P2` 实现近似重复检测，并展示判断依据。

### 6.3 复习与写作工作流

- [ ] `P2` 实现知识点总结和复习提纲。
- [ ] `P2` 实现基于知识库证据的大纲生成。
- [ ] `P2` 引入 LangGraph，实现逐节写作、检查点和失败恢复。
- [ ] `P2` 实现写作内容的引用检查、冲突检查和自动修订建议。
- [ ] `P2` 支持 Word 导出。

## 7. 持续质量任务

- [ ] 为数据库迁移、模型网关、导入管线、混合检索和引用校验建立单元测试。
- [ ] 为四类来源导入、可信问答、索引重建和任务恢复建立集成测试。
- [ ] 建立包含页码、表格、图片 OCR、中文文本和重复内容的固定回归样例集。
- [ ] 记录解析成功率、索引耗时、检索延迟、回答首 Token 延迟和引用校验失败率。
- [ ] 对 Embedding 调用执行批量、限速、重试和用量统计。
- [ ] 增加本地空间占用展示、旧文档版本清理和废弃索引回收。

## 暂不实现

- 多人协作与复杂权限。
- 本地大语言模型推理。
- 移动端应用。
- 插件市场。
- 完整笔记编辑器。
- 复杂知识图谱编辑器。
- 应用关闭后的常驻网页更新服务。
- 任意 OpenAI-compatible API。
